# Workshop: Claude Cowork สำหรับคนทำงาน

**รูปแบบ:** ออนไลน์ผ่าน Zoom

**เวลา:** 20:00 – 23:00 น. (ทุกคืน)
**วิทยากร:** อาจารย์สามิตร
**Platform:** Zoom Meetings + Claude Cowork Desktop App

## วัตถุประสงค์หลักสูตร (Course Objectives)

เมื่อจบ Workshop ผู้เรียนสามารถ:
1. เข้าใจความแตกต่างระหว่าง AI Chatbot ทั่วไปกับ Agentic AI และนำ Claude Cowork ไปใช้งานจริงได้
2. อธิบายความแตกต่างระหว่าง **Claude Cowork** และ **Claude Code** รวมถึงเลือกใช้ให้เหมาะสมกับบทบาท
3. ใช้ Claude ช่วยสร้างเอกสาร Excel / Word / PowerPoint และจัดการข้อมูลได้อย่างมีประสิทธิภาพ
4. วิเคราะห์เอกสาร งบการเงิน และสรุปรายงานธุรกิจด้วย Claude
5. เชื่อมต่อ Claude กับ Google Drive, Gmail, Slack และ Google Calendar ใน Workflow จริง
6. ตั้ง Scheduled Tasks และออกแบบ Automated Workflow ขององค์กร
7. ใช้ AI อย่างปลอดภัย เข้าใจ Prompt Injection และ Data Classification

### 🟦 Module 6 — MCP & Enterprise Connectors (75 นาที)

**เวลา:** 21:15 – 22:30 น.

#### วัตถุประสงค์
- เข้าใจว่า MCP คืออะไร และทำงานอย่างไรในเชิงปฏิบัติ
- เชื่อมต่อ Claude กับ Google Drive, Gmail, Google Calendar และ Slack ได้
- ออกแบบ Workflow โดยใช้หลาย Connector ประกอบกัน

#### หัวข้อ

**ส่วนที่ 1 — MCP คืออะไร? (15 นาที)**
1. **MCP (Model Context Protocol) — แนวคิดหลัก**
   - MCP = สะพานเชื่อม Claude กับ App ภายนอก โดยไม่ต้องเขียน Code
   - Architecture: `Claude ↔ MCP Server (local) ↔ Service API ↔ ข้อมูลจริง`
   - Connector = MCP Server สำเร็จรูปที่ติดตั้งผ่าน GUI ใน Cowork
2. **Token & Permission**
   - Token = "กุญแจ" ที่ Claude ใช้เข้าถึงข้อมูล — ให้ Permission เฉพาะที่ต้องการ
   - Least Privilege: อย่าให้สิทธิ์เกินจำเป็น
   - Scope ตัวอย่าง: `Gmail.Read` ≠ `Gmail.Send` ≠ `Gmail.Delete`
3. **Connectors ที่มีใน Cowork และ Use Cases สำหรับคนทำงาน**

   | Connector | ใช้ทำอะไรได้ | Scope ที่ต้องการ |
   |-----------|------------|-----------------|
   | Google Drive | ค้นหา / อ่าน / สรุปไฟล์ | Drive.Read |
   | Gmail | สรุป / Draft / Triage Email | Gmail.Read, Gmail.Compose |
   | Google Calendar | ดูตาราง / เตรียมประชุม | Calendar.Read |
   | Slack | อ่าน Channel / ส่ง Notification | channels:read, chat:write |

**ส่วนที่ 2 — Setup & Demo ทีละ Connector (45 นาที)**

4. **Google Drive Connector** (10 นาที)
   - Setup: Connect ผ่าน Cowork Settings
   - Demo: "ค้นหาไฟล์ที่แก้ไขล่าสุดและสรุปให้หน่อย"
   - Use Case: ค้นหา Proposal ล่าสุด, รวบรวม Report หลายไฟล์

5. **Gmail Connector** (15 นาที)
   - Setup: Google OAuth ใน Cowork
   - Demo A: Email Triage — "แยก Email เร่งด่วน / ทั่วไปจาก 10 ฉบับล่าสุด"
   - Demo B: Draft Reply — "Draft ตอบ Email ลูกค้าอิงจาก proposal ใน Drive"
   - Use Case: ลด Inbox Zero จาก 2 ชั่วโมง → 20 นาที

6. **Google Calendar Connector** (12 นาที)
   - Setup: Calendar Permission
   - Demo: "วันนี้มีประชุมอะไรบ้าง เตรียม Brief สั้นๆ ให้หน่อย"
   - Use Case: "30 นาทีก่อนประชุม ให้ Claude เตรียมข้อมูลจาก Drive + Email"

7. **Slack Connector** (8 นาที)
   - Setup: Slack App Permission
   - Demo: "ส่งข้อความใน #team-updates ว่า Report พร้อมแล้ว"
   - Use Case: Standup Bot, Meeting Reminder, Project Alert

**ส่วนที่ 3 — Multi-Connector Workflow Demo (15 นาที)**

8. **การรวม Connector หลายตัวในคำสั่งเดียว**
   ```
   "เตรียมฉันสำหรับประชุม 14:00 น. วันนี้:
    1. ดูรายละเอียดใน Calendar
    2. ค้นหาไฟล์ที่เกี่ยวข้องใน Drive
    3. ดู Email thread ล่าสุดของเรื่องนี้
    4. สรุปเป็น Brief 1 หน้า"
   ```
9. **Use Cases จริงแยกตาม Role:**
   - **Finance:** Email แจ้งยอด + Drive งบ → สร้าง Summary Report
   - **Manager:** Calendar ประชุมทั้งสัปดาห์ + Slack → Weekly Priority List
   - **Sales:** Email ลูกค้า + Drive Proposal → Draft Follow-up
   - **HR:** Calendar Interview + Drive CV → เตรียม Interview Brief

## กลุ่มเป้าหมาย (Target Audience)

- **พนักงานออฟฟิศ (Knowledge Worker)** — จัดการเอกสาร ข้อมูล และงาน Routine
- **หัวหน้าทีม / ผู้จัดการ / ผู้บริหาร** — ยกระดับ Productivity ทีม
- **นักบัญชี / ผู้จัดการการเงิน** — วิเคราะห์งบการเงินและสรุปรายงานด้วย AI
- **Freelancer / ที่ปรึกษา / ผู้ประกอบการ** — Scale Output โดยไม่เพิ่มคน
- **ทุกคนที่อยากใช้ AI ในงาน** — ไม่จำเป็นต้องมีพื้นฐานเขียนโค้ด
