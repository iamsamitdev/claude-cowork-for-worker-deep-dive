# Decision Framework: เลือกใช้ Claude Cowork อย่างมีประสิทธิภาพ

## Framework 1: งานไหนเหมาะกับ AI Agent?

ก่อนมอบงานให้ Claude ถามตัวเองว่า:

```
✅ งานนี้เหมาะกับ Claude ถ้า:
   - มีขั้นตอนซ้ำๆ ที่ชัดเจน
   - Input และ Output นิยามได้ชัด
   - ไม่ต้องการ Judgement เชิงจริยธรรมสูง
   - สามารถ Verify ผลลัพธ์ได้ง่าย

⚠️ งานนี้ต้องระวัง:
   - ต้องการ Context ที่ซับซ้อนมาก
   - ผลกระทบสูงถ้าผิดพลาด
   - ต้องการ Human Relationship

❌ งานนี้ไม่ควรให้ Claude ทำคนเดียว:
   - ตัดสินใจทางกฎหมาย / การเงินสำคัญ
   - ส่งสื่อสารกับลูกค้าโดยไม่ Review
   - เข้าถึงข้อมูล Confidential โดยไม่จำเป็น
```

## Framework 2: Prompt Quality Checklist

ก่อน Submit Prompt ให้ Claude ตรวจสอบ:

- [ ] **Task ชัดเจน** — บอกว่าต้องการทำอะไรกันแน่
- [ ] **Context ครบ** — บอก Background ที่จำเป็น
- [ ] **Output Format** — บอกรูปแบบที่ต้องการ (Table, List, Paragraph)
- [ ] **Constraints** — บอกข้อจำกัด เช่น ความยาว, ภาษา, สไตล์
- [ ] **Example** (ถ้ามี) — ให้ตัวอย่าง Output ที่ต้องการ

### ตัวอย่าง Prompt ที่ดี

```
"วิเคราะห์ไฟล์ meeting_transcript_01.txt และ:
1. สรุปประเด็นหลัก 3-5 ข้อ
2. สกัด Action Items ทั้งหมด พร้อมผู้รับผิดชอบและ Deadline
3. ระบุ Decision ที่ตัดสินใจในที่ประชุม

Output format: Markdown Table สำหรับ Action Items
ภาษา: ภาษาไทย
ความยาว: ไม่เกิน 1 หน้า A4"
```

### ตัวอย่าง Prompt ที่ไม่ดี

```
"สรุปการประชุมให้หน่อย"
```
(ไม่มี Context, ไม่มี Format, ไม่รู้ว่าต้องการอะไร)

## Framework 3: Human-in-the-Loop Points

จุดที่ควรให้ Human ตรวจสอบก่อน Proceed:

1. **ก่อนส่ง Email** — Review Draft ทุกครั้ง
2. **ก่อน Delete หรือ Move ไฟล์** — ยืนยันรายการ
3. **ก่อน Publish หรือ Share** — ตรวจสอบเนื้อหา
4. **ก่อนส่งรายงานการเงิน** — Verify ตัวเลขด้วยมือ
5. **หลัง Automation Run** — ตรวจสอบ Output log
